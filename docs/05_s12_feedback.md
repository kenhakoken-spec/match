# S12 殿FB（正典）— 2026-06-03。コンテキスト限界のため確実に記録

殿の大量FB。漏れなく対応する。開発将軍が順次実装→実機確認→デプロイ。

## FB一覧（原文趣旨）と対応方針

1. **性別の二重入力**: 最初(onboarding)で性別を選ぶのに、プロフ登録でもまた選んでいる。→ **プロフ登録から性別選択を削除**（onboardingの値=Profile.genderを正とする。重複排除）。

2. **未ログインで男性の優良(参加費)が見えるのは違う**: 公開プレビュー(/explore)で料金を出している。→ **未ログインの公開プレビューでは料金を出さない**（または「参加費あり」程度。性別別の金額は出さない）。

3. **「参加費」と明記**: ¥2,000は「参加費」と書く。**飲み代は別**だと明示（「参加費 ¥2,000（飲食代は別途）」等）。

4. **ひとこと自己紹介(bio)の表示先**: マッチング(飲み会)**確定時=成立詳細(/matches/[id])で見える**ようにする。一覧では出さない。

5. **会の確定タイミング**: **前日の夜(例 前日19時)に確定**でよい。会場・予約情報の通知もそのタイミング想定。仕様/文言に反映。

6. **職業はフリー入力**: occupation を**自由入力(フリーテキスト)**に（現状enum選択）。**マッチ時=成立詳細で表示**。

7. **年齢を表示**: **成立詳細で年齢**(生年月日から算出)を出す。

8. **写真は登録不要・アイコン選択に**: 写真アップロード廃止。**アイコン(プリセット)を選ぶだけ**。本人確認の身分証写真は別途必要(これは残す)。

9. **TOPコンセプト変更**: 「写真詐欺の不安もない」コピーは**不要**(写真自体使わないため)。コンセプトは**「その場の新たな出会い」**を主役に。LP/価値訴求を見直す。

10. **定員は3:3でなくても2:4も可**: 男女厳密3:3固定でなく、**合計6人で柔軟(2:4等も最悪OK)**。成立条件・定員ロジック・表示を緩める。

11. **枠詳細プレビューが見れない(バグ)**: /explore/[id] が「会が見つかりません」。公開プレビュー詳細のデータ取得/FALLBACKを直す。

12. **料金コピーの「あなた=女性」がAIっぽい**: 自然な文言に。"あなた=女性"のような説明をやめる。

13. **決済ができていない?**: 決済フローの状態を確認・整備（モック確定でも動線が通るように）。

14. **成立詳細に職業・年齢を出す**: /matches/[id] のメンバー表示に occupation と age を追加(現状 displayName+gender のみ)。

15. **予約詳細の入力口**: 会場・予約名・予約URL・集合場所を**運営(admin)が入れるUI**を用意(入れる口)。誰がどう入れるか動線を作る。

16. **評価の期限**: 相互評価は**翌日の夜までに**入れてもらう。期限の文言/設計。

17. **運営ADMINが見れない**: 本番で殿がadmin画面を見られない(307でブロック)。**殿のLINE userIdをadmin化する手段**(ADMIN_LINE_USER_IDS env 等)を整備し、運営として閲覧・操作できるように。
