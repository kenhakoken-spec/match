## 0. ブランド刷新（最重要・全画面横断）

- **新名称: `HAKO-NIWA`（主）/ `箱庭`（副）**。旧 `rendez` を全置換。
  - ロゴ・ヘッダは `HAKO-NIWA`、日本語文脈で「箱庭」を併記。
  - タグライン: **「みんなが出会える場所」**。
  - 由来の表現: 箱庭＝小さくて安心できる、手入れの行き届いた庭。そこに人が集い、自然に出会う。
- **コンセプトは少し幅を広げる**: 「恋愛の合コン」一点突破でなく、
  **「安心できる場で、人と出会う」**を上位概念に。実装の主機能は現行どおり
  **男女3対3＝6人で会う**を中心に据えつつ、文言は「出会い・つながり」を含む広めのトーン。
  誇大はしない（design-system §8 維持）。
- 反映先: `src/app/layout.tsx`(metadata) / LoginScreen / ComingSoon / coming-soon /
  README / docs全般 / 規約類。`rendez` の残存ゼロにする（テスト内の固有名は影響範囲を見て対応）。

## 1. 性別を最初に入力（料金UIに直結）★殿の追加要件

- **オンボーディングの早い段階で性別を入力**させる（現状はプロフィール登録 U-02 まで性別未確定）。
  理由: 料金が性別で変わる（男性 ¥2,000・女性無料）ため、**枠一覧/詳細の料金表示が性別に依存**する。
- **異性側の金額は見せない**:
  - **女性**: 料金（¥2,000）を**表示しない**。「女性は無料です」だけ、もしくは料金行ごと出さない。
  - **男性**: 「¥2,000（初回無料）」を明示。
- 実装方針:
  - 公開プレビュー（未ログイン /explore）は性別不明なので「男性 ¥2,000 / 女性 無料」を**中立的に併記**でよい（現行どおり）。
  - **ログイン後**は、セッションの性別に応じて料金表示を出し分け（女性には男性料金を出さない）。
  - 性別の早期入力: onboarding に「あなたの性別」ステップを追加 or LINEログイン直後にプロフィール最小入力（性別）へ誘導。
  - 性別は Profile.gender が正。途中入力したものを Profile 作成時に引き継ぐ。

## 2. フロー/導線の作り込み（撮影後しか枠が見えない問題）

現状: onboarding → /identity（撮影）→ /identity/status → /profile/new → /browse。
**「撮影しないと枠が見えない」「撮影後ホームで予約できるか不明」**を解消する。

- **見える化**: 枠一覧/詳細は **未認証でも /explore で閲覧可**（S8で実装済）。これを導線上で明示し、
  「まず見てから登録・本人確認」へ自然につなぐ。onboarding/各画面に「会を見てみる」動線を置く。
- **認証後の予約可否を明確に**: /browse（ホーム）や枠詳細で、
  「本人確認 承認済か / プロフィール有るか / 応募できるか」を**ステータスとして明示**（できない理由を淡色で）。
- **撮影後の着地**: /identity/status の「承認待ち」でも、`/explore` や `/browse` を見られると明示。
  「審査中でも会の一覧は見られます」等の文言で迷子を防ぐ。
- **本人確認の通知の行き先**: 承認/却下は NotificationLog（type=identity_approved/rejected）に記録され、
  本番は LINE Messaging で本人へ。現状は通知ログ＋ /identity/status で確認。**画面に「結果はLINEとこの画面で通知」と明記**。

## 3. UIの謎文言・画像欠落

- **onboarding**: 画像（イラスト）が無いまま枠だけのスライドで「次へ」が浮く問題。
  → **編集的なSVG/装飾**（箱庭＝小さな庭のミニマルな線画）を各スライドに置く or レイアウトを画像無し前提で
  きれいに再設計（余白・タイポで魅せる）。「次へ」ボタンの文脈を明確に。
- **「次へ」周辺の文言**: 各画面のボタン文言を、次に何が起きるか分かる具体語に（例「会を見てみる」「本人確認へ進む」）。
- 画像が必要な箇所は **public/ に最小のSVGを置く**（外部依存なし・軽量・編集的トーン）。
