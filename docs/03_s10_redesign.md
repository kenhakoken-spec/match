# S10 全面リデザイン（正典）— ペルソナ/マーケ起点で「最高のプロダクト」へ

殿の指示（2026-06-02）: 「LINEで始めるを押すとエラー / 最初の男女登録が無く結局見えている / 画像っぽいものが無く見づらい / 今の画像も見づらい。**もう一度全部見直して、ペルソナ・マーケティングの観点から最高のプロダクトを目指して改修しきって**」。

開発将軍が実機（本番 match-nomi.vercel.app）をスクショ確認した上での棚卸しと、改修方針。

## A. 実機確認で判明した具体的問題（私の目視）

1. **箱庭SVG（庭の俯瞰線画）が意味不明で逆効果**
   - LP・onboarding上部に〇や↑が並ぶ謎の線画。何を表すか伝わらず、安っぽく見える。**撤去 or 全面刷新**。
2. **LPの固定フッタCTAがヒーロー本文を隠す**（レイアウト崩れ）
   - 「LINEではじめる/会を見てみる」の固定パネルが画面中央に居座り、「男女3人ずつ…」本文に被る。
   - → LPは**スクロール型の通常フロー**にし、CTAは適切な位置に。固定するなら本文を隠さない設計。
3. **ビジュアルが弱い**（画像っぽいものが無い）
   - アイコン羅列で平板。ヒーローに世界観が伝わるビジュアル（質の高い抽象グラフィック/写真風）が必要。
4. **LINEではじめるでエラー**（殿報告）
   - 技術的にはLINEログイン遷移は正常（access.line.me へ正しく飛ぶ）。スマホLINE内なら通るはずだが、
     殿の環境で失敗する。**エラー時のUX**（何が起きたか・次にどうするか）と、**LINE外で開いた場合の案内**を改善。
   - 「最初の男女登録が無く結局見えている」＝ログインで詰まり性別登録に進めない→未ログインのexploreだけ見えている状態。ログイン導線を確実に。

## B. ペルソナ（再確認・LPの軸）

- **コア**: 20代後半〜30代後半、東京（恵比寿/池袋/銀座勤務 or 在住）。
- **状況**: 1対1マッチングアプリに疲れた／時間を無駄にしたくない／会う前のメッセージ往復が面倒／安全・本人確認を重視／自然な出会いを求める。
- **不安**: 「変な人が来ないか」「写真詐欺」「1対1は気まずい」「ドタキャン」。
- **刺さる価値**: ①3対3グループ＝気まずさ・危険が薄まる ②全員本人確認済 ③チャット不要、会うことに集中 ④会場おまかせ ⑤評価/バッジで質担保 ⑥ドタキャン罰金で真剣な人だけ。

## C. マーケティング観点の改修方針（LP）

- **ファーストビューで価値が刺さる**: タグライン＋主見出し＋「誰のための/何が違うか」を3秒で。
- **不安の解消を前面に**: 「本人確認必須」「3対3だから安心」「真剣な人だけ（ドタキャン罰金）」をベネフィット言語で。
- **世界観ビジュアル**: 「箱庭＝安心して人と出会える小さな庭」を、**洗練された抽象グラフィック**で表現（CSS グラデ/有機的シェイプ/上質な余白）。謎の線画はやめる。画像が無くても**CSSで作る上質なビジュアル**で成立させる（外部画像依存を避けつつ「画像っぽい」リッチさを出す）。
- **社会的証明・具体性**: 「恵比寿/池袋/銀座で毎週」「水・金・土 19:30」等の具体で実在感。
- **CTA設計**: 主CTA「LINEではじめる」、副「まず会を見てみる」。スクロール各所に自然に配置。固定バーにするなら半透明で本文を隠さない高さに。
- **離脱対策**: LINE外で開いた人向けに「スマホのLINEで開いてね」の案内（QR/リンク）も検討。

## D. 改修スコープ（画面別）

1. **LP（`/` LoginScreen）**: 全面リデザイン。ヒーロー（CSSビジュアル）→不安解消の価値→使い方→具体（エリア/曜日）→CTA。固定CTA被り解消。
2. **`/coming-soon`（ComingSoon）**: 同じ世界観で「近日公開」LP。
3. **onboarding**: 謎SVG撤去、上質なビジュアル/余白に。性別ステップは維持しつつ洗練。
4. **箱庭ビジュアル**: 謎の庭線画SVGを廃止し、CSSベースの抽象グラフィック（or 上質な幾何/有機シェイプ）に差し替え。`BrandMotif` 作り直し。
5. **ログインUX**: エラー時の文言・再試行・LINE外案内。LINEログイン遷移自体は維持。
6. **explore/browse等の各画面**: LP世界観と整合する微調整（フォント/余白/色）。料金出し分け（S9実装済）は維持。
