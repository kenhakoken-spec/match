## 0. デザイン原則（最上位）— 「AIっぽくない」の言語化

master_plan の世界観は **「"AIっぽさ"を排した、温かく編集的（editorial）」**。
これは「雑誌・上質な飲食店の案内・落ち着いた紙もの」に近い質感を指す。下記を判断基準とする。

### 避ける（= 量産SaaS / AIっぽい）
- **紫〜青のグラデーション**、ネオン、彩度の高いビビッドな原色。
- **絵文字の多用**、顔文字、過剰なアイコン装飾。
- 角丸を大きく取りすぎた「ぷっくり」したカード／ボタンの量産感。
- どこかで見た **テンプレSaaS** 構図（巨大ヒーロー + 3カラム特徴 + キラキラ）。
- 強い影でフワフワ浮かせる **マテリアル過多**。
- マーケ風の煽りコピー（「運命の出会い！」「今すぐ無料登録！」「業界No.1」）。
- 中身の根拠がない装飾・イラストの飾り付け。

### 目指す（= 温かく編集的）
- **生成り・オフホワイトの紙のような地** に、**インクのような濃い文字**。
- アクセントは **テラコッタ / 焦がしオレンジ**（食事・人の温度を想起）を**1色だけ**主役に。
- **写真を主役**にできる余白とフレーミング。写真は自然光・等身大（加工しすぎない）。
- **明朝（serif）見出し + ゴシック（sans）本文** の対比で、編集物の品をつくる。
- 影は最小限、境界は **細いボーダー** か **わずかな影** で軽く。
- コピーは**機能的で誠実**。落ち着いた敬語、誇張しない（master_plan「簡易・軽量。盛りすぎない」）。
- 余白は広め。詰め込まず「読み物」として呼吸させる。

> 一言基準: 「これは雑誌の特集ページに置けるか？ SaaS のLPっぽくないか？」を毎回問う。

## 1. カラーパレット

彩度を抑えた温かいニュートラル + テラコッタ1アクセント。紫・原色ブルーは不使用。

### 1.1 ベース（地・面）
| トークン | Hex | 用途 |
|----------|-----|------|
| `bg/base` | `#FBF7F0` | アプリ全体の地（生成りのオフホワイト） |
| `bg/surface` | `#FFFFFF` | カード・入力欄など最前面の面 |
| `bg/sunken` | `#F2ECE2` | くぼみ・セクション区切り・タブ非選択地 |
| `bg/overlay` | `rgba(43,38,34,0.45)` | モーダル背後のスクリム |

### 1.2 インク（文字・線）
| トークン | Hex | 用途 |
|----------|-----|------|
| `ink/900` | `#2B2622` | 主見出し・本文（黒に寄せた焦げ茶。純黒は使わない） |
| `ink/700` | `#4A433C` | 通常本文 |
| `ink/500` | `#7A7066` | 補助テキスト・キャプション |
| `ink/300` | `#B8AE9F` | プレースホルダ・非活性文字 |
| `line/200` | `#E5DCCD` | 通常ボーダー・区切り線 |
| `line/100` | `#EFE8DC` | 極薄の区切り |

### 1.3 アクセント（主役 = テラコッタ）
| トークン | Hex | 用途 |
|----------|-----|------|
| `accent/600` | `#A85638` | プライマリのホバー / 押下・白文字を載せる地 |
| `accent/500` | `#C2703D` | **プライマリ**（主要ボタン・選択・リンク強調） |
| `accent/300` | `#E7B79A` | 選択中の淡い面・チップ選択地 |
| `accent/100` | `#F6E7DC` | 最も淡い強調背景（バナー地など） |

### 1.4 セカンダリ（補助の差し色 = 深い緑）
食事・落ち着きを補強する低彩度のグリーン。多用しない（成功 / 穏当な状態などに限定）。
| トークン | Hex | 用途 |
|----------|-----|------|
| `secondary/500` | `#5E7A57` | 補助アクセント・「成立」など穏当な強調 |
| `secondary/100` | `#E7EDE6` | その淡い面 |

### 1.4b バッジ / 信頼系（本人確認済 / 優良バッジ）
"勲章感"を出しすぎない、くすんだ真鍮（ブラス）系。金ピカ・グラデ・キラキラは不可（量産SaaS回避）。
| トークン | Hex | 用途 |
|----------|-----|------|
| `trust/600` | `#8A6D3B` | バッジの文字 / アイコン線（くすんだ真鍮） |
| `trust/300` | `#D8C39A` | バッジの枠線 |
| `trust/100` | `#F3EAD8` | バッジの淡い地（pill 背景） |
| `verified/500` | `#5E7A57` | 「本人確認済」チェックは控えめに緑（= secondary/500。信頼=穏当） |

- **優良バッジ** = `trust` 系（真鍮の小さな pill / 線アイコン）。`accent` は使わず、主役（応募ボタン等）と競合させない。
- **本人確認済** = `verified/500` の小さなチェック + ラベル。装飾は最小（安心は静かに伝える）。

### 1.5 状態色（ステータス）
彩度を抑え、地になじませる。応募ステータス（[`screen-flow.md`](./screen-flow.md) §2.4）に対応。
| 状態 | トークン | Hex | 対応ステータス |
|------|----------|-----|----------------|
| 情報 / 募集中 | `state/info` | `#5B7186` | 募集中（くすんだ青グレー。原色ブルー回避） |
| 成功 / 成立・会場確定 | `state/success` | `#5E7A57` | 成立 / 会場確定（= secondary/500） |
| 注意 / 締切間近 | `state/warn` | `#C08A2E` | まもなく締切・前日 |
| 不成立 / 終了 | `state/muted` | `#9A9082` | 不成立・キャンセル・締切（グレー） |
| エラー | `state/danger` | `#B0463C` | 通信エラー・破壊的操作 |

### 1.6 コントラスト基準（アクセシビリティ）
WCAG AA を最低ラインとする（master_plan の信頼感・役割の品質基準に基づく）。
- 本文（`ink/700` on `bg/base`）: コントラスト比 **7:1 以上**（AAA 相当）。
- プライマリボタン文字（白 on `accent`）: **4.5:1 以上**を満たすこと。`accent/500 #C2703D` で白文字が 4.5:1 に届かない使用箇所（小さい文字等）は **背景を `accent/600 #A85638`** に上げる。大きいラベル（label/h2 相当）は `accent/500` 可。
- 非活性でも `ink/300` は **3:1 以上**を死守（完全に読めない灰色にしない）。
- 状態色は**色のみに依存しない**（必ずラベル文字 or アイコン形状を併記。色覚多様性対応）。

## 2. タイポグラフィ

編集物の品をつくる「明朝見出し × ゴシック本文」。日本語が主・英数混在。

### 2.1 フォントファミリー
| 役割 | フォント（推奨 / フォールバック） | 備考 |
|------|-----------------------------------|------|
| 見出し（serif） | `"Shippori Mincho", "Noto Serif JP", serif` | 明朝。タイトル・成立詳細の日時等の「編集感」担当 |
| 本文 / UI（sans） | `"Noto Sans JP", "Hiragino Sans", system-ui, sans-serif` | 可読性重視。ボタン・フォーム・本文 |
| 数字・日時（任意） | sans のまま tabular numerals | 充足ドット横の「あと1名」など桁ぞろえ |

> 注: 見出しの明朝は「ここぞ」で使う。全部を明朝にしない（読みづらさ・古さを避ける）。本文は必ず sans。

### 2.2 タイプスケール（モバイル基準 / px・行間）
| トークン | size / line-height | weight | 用途 |
|----------|--------------------|--------|------|
| `display` | 28 / 1.35 | serif 600 | オンボの大見出し・成立詳細の日時 |
| `h1` | 22 / 1.4 | serif 600 | 画面タイトル（編集感を出す箇所） |
| `h2` | 18 / 1.5 | sans 700 | セクション見出し |
| `body-lg` | 16 / 1.7 | sans 400 | 本文（標準） |
| `body` | 14 / 1.7 | sans 400 | 補助本文・カード内テキスト |
| `caption` | 12 / 1.6 | sans 400 | キャプション・注記 |
| `label` | 13 / 1.4 | sans 600 | ボタン・チップ・タブのラベル |

- 本文行間は広め（1.7）で「読み物」感を出す。
- 最小可読サイズは 12px（`caption`）。これ未満を本文用途に使わない。
- 字間（letter-spacing）: 見出し serif はわずかに詰める（-0.01em）、ラベルはわずかに広げる（+0.02em）程度。やりすぎない。

## 3. 余白・角丸・影・グリッド

「呼吸する余白」「最小の影」「控えめな角丸」が編集感の要。

### 3.1 スペーシングスケール（8 基準、4 補助）
`4 / 8 / 12 / 16 / 24 / 32 / 48`（px）。原則 8 の倍数、微調整に 4。
- 画面左右の標準マージン: **20px**（スマホ縦で詰まりすぎず広すぎず）。
- カード内パディング: **16px**。
- セクション間: **24〜32px**。
- 要素を詰め込まない。迷ったら **広い方**を選ぶ（編集的余白）。

### 3.2 角丸（radius）
ぷっくりさせない。控えめ一貫。
| トークン | 値 | 用途 |
|----------|----|------|
| `radius/sm` | 6px | チップ・タグ・入力欄 |
| `radius/md` | 10px | カード・ボタン |
| `radius/lg` | 14px | モーダル・大きな写真フレーム |
| `radius/full` | 999px | アバター・ドット・トグル |

> 18px 以上の大角丸は使わない（SaaS の「ぷっくり量産感」回避）。

### 3.3 影（elevation）
影は最小限。基本は**ボーダーで分離**し、浮かせる必要がある時だけ薄い影。
| トークン | 値 | 用途 |
|----------|----|------|
| `shadow/none` | none（`1px solid line/200` で代替） | 通常カード（推奨はこちら） |
| `shadow/sm` | `0 1px 2px rgba(43,38,34,0.06)` | わずかな浮き（押下可能カード） |
| `shadow/md` | `0 6px 20px rgba(43,38,34,0.10)` | 固定フッタ・モーダル・トースト |

> マテリアル風の強い影・色付き影は不可。

### 3.4 グリッド / 構成
- スマホ縦は基本 **1 カラム**。カードを縦に積む。
- 充足ドット等の細かな並びは内部で横並び。ページ全体は単一カラムを崩さない。
- **構成優先**: 各画面は「主役（写真 or 主要アクション）→ 補助情報 → 操作」の縦の重みづけを先に決め、要素の羅列にしない（役割の品質基準）。
- 固定フッタ（主要アクション）は `bg/surface` + `shadow/md` + 上ボーダーで本文から分離。

## 4. 主要コンポーネント規約

### 4.1 ボタン
| 種別 | 見た目 | 用途 |
|------|--------|------|
| Primary | 地 `accent/500`（小文字は `accent/600`）/ 文字 白 / `radius/md` / 高さ 52px / `label` | 画面の主アクション（応募する・登録する・送信） |
| Secondary | 地 透明 / 線 `1px line/200` / 文字 `ink/700` | 副次（編集する・あとで） |
| Text | 文字 `accent/500` のみ | 弱い操作（スキップ・やめる・リンク） |
| Danger(text) | 文字 `state/danger` | 取り消し・退会など破壊的 |

- 高さは指で押しやすい **最低 48px**（タップターゲット 44×44pt 以上を厳守 / アクセシビリティ）。
- **非活性**: 地 `bg/sunken` / 文字 `ink/300` / カーソル不可。必須未充足時に使用（例: オンボ同意+年齢前、応募確認の両チェック前）。
- 1 画面のプライマリは原則 **1 つ**（固定フッタに置く）。
- ラベルは動詞 + 目的語で具体的に（「応募する」>「OK」）。

### 4.2 カード（枠カード / 応募ステータスカード）
- 地 `bg/surface`、`1px solid line/200`、`radius/md`、パディング 16px、影なし基本。
- 枠カード構成: **エリア・日時（上）→ 充足ドット（中）→ 残数・締切（下）→ 詳細ボタン（右下）**。
- 充足は **ドット表現**（`●`=確定 / `○`=空き）＋テキスト「あと女性1」。色のみに頼らない。
- ステータスバッジは左肩に。色 + ラベル文字（成立 / 募集中 / 不成立）を必ず併記。
- 押下可能カード全体をタップ領域にする場合のみ `shadow/sm`。

### 4.3 フォーム（プロフィール登録 等）
- ラベルは入力欄の**上**に常時表示（プレースホルダだけに頼らない / アクセシビリティ）。
- 入力欄: 地 `bg/surface`、`1px line/200`、`radius/sm`、高さ 48px、フォーカスで枠 `accent/500`。
- プレースホルダ: `ink/300`。補足は欄の下に `caption`。
- 文字数カウンタ（自己紹介など）は右下に `caption`。
- エラー: 枠 `state/danger` + 欄下に赤文字メッセージ（理由を具体的に）。
- 性別は 3対3 判定の根幹のため必須・2択を明確に（[`wireframes.md`](./wireframes.md) U-02）。
- 写真アップロードは最上位。1 枚目=メインを明示。写真ガイド(U-02b)への `ⓘ` 導線を付す。
- 生年月日入力は年齢確認（18歳以上 / master_plan §8）に直結 → 未成年は次へ進めない制御を前提に置く。

### 4.4 ナビゲーション
- **下部タブ 3 つ**（枠をさがす / 応募状況 / マイページ）。地 `bg/surface` + 上ボーダー。
- 選択タブ: アイコン+ラベルを `accent/500`、非選択: `ink/500`。塗りつぶしバッジ等で過剰装飾しない。
- ヘッダ: 画面タイトル（`h1`、必要箇所のみ serif）+ 戻る `←`。透明 or `bg/base`。
- LINE プッシュ通知からの深いリンク着地を前提に、各画面は単独でも文脈が分かる見出しを持つ。

### 4.5 通知（LINE プッシュ）の文面トーン
画面ではないが世界観の一部。チャット無しの本サービスでは通知が主要接点。
- 件名は事実ベース・落ち着き: 「成立しました」「会場が決まりました」「明日です」。
- 会場通知は master_plan §3-7 の要素（日時 / エリア / 店名 / 予約URL / 予約名 / 集合場所）を簡潔に。
- 煽らない・絵文字は最大1つまで。
- 必ず「タップで詳細（U-08）」に着地。

### 4.6 アイコン / イラスト / 写真
- アイコンは**線画（stroke）**で統一、塗り多用しない。太さ 1.5〜2px。
- 装飾イラストは最小限。使うなら**手描き感のある編集的トーン**（フラットなSaaSイラストや3Dレンダの多用は避ける）。
- 写真は自然光・等身大・適度な余白でフレーミング。過度なフィルタ / 彩度上げをしない（写真ガイドU-02bと一貫）。

### 4.7 追加コンポーネント規約（本人認証 / 限定条件 / 決済 / 評価 / バッジ）

今回の要件追加で必要な5系統。いずれも §0 の方針（編集的・抑制）を崩さないこと。
対応画面は [`wireframes.md`](./wireframes.md) / [`screen-flow.md`](./screen-flow.md) 参照。

#### A. 参加条件チップ（20代限定 / 優良バッジ限定）— U-04 / U-05 / A-02
- 形: 小さな pill。`radius/sm`、高さ 22〜24px、`caption`(12) or `label`(13)、左右パディング 8px。
- 見た目: 地 `bg/sunken` / 線 `line/200` / 文字 `ink/700`。**色で煽らない**（条件は事実情報）。表記は「〔20代限定〕」「〔優良バッジ限定〕」。
- 優良バッジ限定チップのみ、頭に `trust` 系の小アイコン `◆` を付けてバッジと視覚連結。
- **条件を満たさない枠**: カード全体を淡色化（不透明度を下げる or `bg/sunken` 地）＋境界を破線（dashed `line/200`）。中に理由を `state/muted` で 1 行（「あなたは対象外です」「この枠は20代限定です」）。**赤=エラー扱いにしない**（落ち度ではないため）。

#### B. 本人認証ステータス表示 — U-12 / U-13 / マイページ / A-09
- ステータスは **色 + ラベル + 形状** の3点で表現（色覚多様性 / §1.6）。
  - 審査中(pending): `state/info` + ラベル「確認中」+ 形状 `◷`（時計）。
  - 承認(approved): `verified/500` + ラベル「確認済」+ 形状 チェック。
  - 却下(rejected): `state/warn`（赤=`danger`にしない。責めず再提出を促す）+ ラベル「要再提出」+ 形状 `⚠`。
- **却下画面は理由を必ず提示**し、再提出ボタンを `primary` で 1 つ。文言は事実ベース・非難しない（「画像が不鮮明でした」）。
- 身分証アップロード枠: 破線 `line/200` のドロップゾーン + カメラ/選択。**PII 配慮メッセージ**（「確認後に削除します」「公開されません」）を `caption` で必ず添える（信頼感 / master_plan §8/§9）。
- admin 審査(A-09)の身分証画像はアクセス制限。承認操作後は **「削除済 ✓」を画面に明示**（偽の成功報告禁止 / master_plan §11）。

#### C. 決済 UI（¥2,000 / 初回無料）— U-14 / U-05 / U-06 / U-07
- **金額表示**: `display` or `h1` 相当で「¥2,000」。通貨は円記号 + 半角数字、tabular numerals。内訳（参加費 / 合計）を `body`。
- **初回無料の見せ方**（最重要）: 初回対象者には **「初回は無料です」を主役**にし、金額より大きく扱う。地 `accent/100`、文字 `accent/600`、絵文字 🎁 は**1つまで**（祝意の例外運用 / §8 と整合）。次回以降 ¥2,000 を `caption` で誠実に予告。
- **女性**: 「無料」をはっきり。割引・煽り表現にしない（「女性無料！」のような販促トーン禁止）。
- **タイミングの明示**: 「成立後にお支払い」「不成立なら課金なし」を必ず併記（誤解・不信の防止）。
- **Stripe 導線**: カード入力は Stripe Elements/Checkout に委譲。アプリ側でカード値を保持しない旨を `caption`（「決済は Stripe で安全に処理されます」）。3DS 等のリダイレクトは Stripe UI に任せる。
- **決済失敗**: `state/danger` のメッセージ + リトライ(`primary`) + 別カード(`secondary`)。理由は具体的に。
- 要決済の応募(U-07)は accent でカードを目立たせ、期限（`state/warn`）を併記。

#### D. 星評価（相互評価）— U-15 / マイページ / admin
- 星: `★`(評価あり) / `☆`(なし)。塗りは `accent/500`（温度のある主役色）。5 段階。サイズはタップしやすく（1つ 32〜40px / ターゲット 44pt）。
- 各被評価者は **カード 1 枚**（◯アバター + 名前 + 5段階）。同席者ぶん縦に積む。
- **任意・匿名性の明示**: 「任意です」「相手に個別開示されません」を `caption` で先頭に（安心して正直に付けられるように）。
- マイページの評価サマリ: 「★ 4.3（12件）」のように **平均 + 件数**を `label`/`body`。星は小サイズの装飾に留め、煽らない。
- 評価は誠実なトーン。ランキングや競争を煽る表現は不可（§8）。

#### E. 優良バッジ — マイページ / U-08 メンバー表示 / A-10
- 見た目: `trust` 系の小さな pill（地 `trust/100` / 線 `trust/300` / 文字 `trust/600`）+ 線アイコン `◆`。**金ピカ・グラデ・光彩エフェクト禁止**（量産SaaS / AIっぽさ回避 / §8）。
- 配置: プロフィール名の近くに **静かに**。サイズは控えめ（高さ 22〜24px）。複数バッジ（◆認証済 / ◆優良）は横並び。
- **未取得ユーザーの進捗表示**（マイページ）: 「高評価の参加 3/5 回」のように事実の進捗を `caption` + ドット。**達成を煽らない**・FOMO 演出をしない（信頼感重視）。
- 成立詳細(U-08)のメンバー一覧では、名前の後に小さく付すのみ（一覧の可読性優先）。

## 8. やってはいけないこと（最終チェック / FAIL 回避）

- 紫グラデ・ネオン・原色ブルー / ビビッド原色を使う → ✗
- 絵文字を散りばめる・顔文字 → ✗（祝意は控えめ1点まで）
- 18px超の大角丸 + 強い影で「ぷっくりSaaS」化 → ✗
- 巨大ヒーロー + 3カラム特徴の量産LP構図 → ✗
- 煽りマーケコピー（運命の / 今すぐ無料 / 業界No.1）→ ✗
- 全文明朝で古臭く・読みにくく → ✗（見出しのみ serif、本文 sans）
- 状態を色だけで表現（ラベル / 形状の併記なし）→ ✗
- 中身の根拠がない装飾・スタンプ的イラストの飾り → ✗

今回の追加要件まわりの NG（§4.7 と整合）:
- 優良バッジを **金ピカ・グラデ・光彩**で勲章演出 → ✗（くすんだ真鍮 `trust` 系で静かに）
- 決済で **販促トーン**（「女性無料！」「今だけ初回0円!!」の煽り）→ ✗（事実を誠実に。初回無料は静かに主役）
- 本人認証の却下を **赤エラー**でユーザーを責める表現 → ✗（`state/warn` + 再提出導線）
- 身分証アップロードで **PII の取り扱い説明を省く** → ✗（削除・非公開を必ず明示）
- 参加条件NGの枠を **赤(danger)で「ダメ」表現** → ✗（淡色 + 事実の理由。落ち度ではない）
- 評価で **ランキング / 競争を煽る** 演出・FOMO → ✗（誠実・任意・匿名性の明示）

> 迷ったら原則 §0 に戻る：「雑誌の特集に置けるか。SaaS/AI っぽくないか。」
