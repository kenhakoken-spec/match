matching-app は「合コン型グループマッチング on LINE（LIFF）」。設計に効く確定前提（2026-05-30 殿の追加指示で大幅更新済み）:

- 男女 **3対3 = 6人**。東京の **恵比寿 / 池袋 / 銀座** の3エリア限定（初期）。
- **アプリ内チャット無し**（一切作らない）。ユーザー間の連絡手段はゼロ → 連絡は全て LINE プッシュ通知 + 成立詳細画面に集約。画面設計の最大の制約。
- **会場は運営が手動手配**（店名 / 予約URL / 予約名 を入力）して6人へ通知。会場DB・予約連携なし。admin に「会場入力&通知送信」画面が必須。
- マッチング方式: **運営が枠＝「日時×エリア」を用意し、ユーザーは応募のみ**（自由検索ではない）。
- **本人認証＝必須**（追加）: 公的身分証アップロード→運営目視確認→承認。**年齢確認(18+)を兼ねる**。**未認証は枠応募不可**（ゲーティング）。身分証画像はアクセス制限・**承認後に削除**（PII最小化）。却下→再提出フローあり。
- **決済 Stripe（追加）**: **男性=1回¥2,000、初回参加は無料**。**女性は常に無料**。**不成立時は課金しない**。成立後に決済 → 全員確定で会場手配へ。カード値は自前保持しない（Stripe委譲）。
- **評価・バッジ（追加）**: イベント後に参加者を相互評価（任意・相手に個別開示しない）→ 高評価が複数回蓄積で**優良バッジ**付与。マイページに評価サマリ+バッジ表示。
- **限定イベント（追加）**: 枠に参加条件（**20代限定** / **優良バッジ限定**）を設定可能。条件不足は応募不可UI（赤エラーにせず淡色+理由）。
- ターゲットは **20代後半〜30代後半**。**スマホ縦が主（LIFF）**。デスクトップは admin のみ。
- コアループは **0〜9 の10ステップ**: 0.ログイン+本人認証 / 1.プロフィール / 2.枠一覧 / 3.応募(条件) / 4.成立 / 5.決済(男¥2000・初回無料/女無料) / 6.会場決定 / 7.6人へ通知 / 8.当日集合 / 9.相互評価→バッジ。
- スタック: Next.js(App Router)+TS+Tailwind / Vercel / LIFF / LINE Messaging API / **Stripe** / Vercel Postgres(Neon)+Prisma / Vercel Blob(身分証はアクセス制限)。
- セキュリティ最優先: 認証/プロフィール/決済/通知の変更後は security-reviewer 必須。PII最小権限・暗号化・最小保持。

**Why:** 正典 `docs/00_master_plan.md`（殿承認済み 2026-05-30、追加指示反映版）。design-worker が全画面を `docs/design/` に定義: **ユーザー17画面(U-00〜U-15・U-E)+ admin 11画面(A-00〜A-10)**。新規= U-12本人認証/U-13審査中/U-14決済/U-15評価/A-09本人認証審査/A-10バッジ付与状況。
**How to apply:** 設計・レビュー時はこの前提に矛盾しないか必ず照合。特に「チャット無し」「会場は運営手配」「未認証は応募不可」「不成立は非課金」「男初回無料/女無料」を崩さない。最新は master_plan を再読して確認（仕様は更新されうる）。関連: [[feedback-design-direction]]
