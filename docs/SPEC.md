## 3. 各種制限（ゲート）— ここが仕様の肝

### ① 本人認証ゲート（必須）
- **未認証（承認前）は枠応募 不可**。U-05 の「応募する」は本人確認 承認済でのみ活性。
- 本人認証＝公的身分証アップロード→**運営目視**→承認。これで **年齢確認(18歳以上) を兼ねる**。
- 身分証画像は**承認/却下後に削除**（PII最小化。`imageDeletedAt` 記録）。
- 法令: 「インターネット異性紹介事業（出会い系規制法）」該当の可能性 → 年齢確認を内包。

### ② 参加条件ゲート（限定イベント）
- 枠ごとに条件設定可能:
  - **20代限定**: `minAge`/`maxAge`（例 20〜29）。生年月日から年齢算出して判定。
  - **優良バッジ限定**: `requiresBadge` → 優良バッジ保有者のみ応募可。
- 条件不足の枠は **応募不可UI**（淡色＋事実の理由。赤エラーにしない）。
- 判定は**サーバ側で再検証**（クライアント表示を信用しない）。応募理由コード: `identity_required / profile_required / age_out_of_range / badge_required / gender_full / already_applied / slot_closed`。

### ③ 決済ゲート（男性のみ・成立後）
- **女性: 常に無料** / **男性: 初回参加は無料、2回目以降 ¥2,000/回**。
- 「初回」= 過去の成立参加が0回。
- **不成立なら課金しない**（成立確定後にのみ確定課金）。金額は**サーバ算出**（クライアントは金額を渡せない）。
- カード情報は**自前保持しない**（Stripeに委譲）。

### ④ 二重防止・原子性
- 同一枠への二重応募禁止（unique）。男女各3名の定員超過を原子的に防止。
- 同一イベントの同一相手への二重評価禁止。自己評価不可。
- 優良バッジは冪等付与（1人1つ）。

## 4. ビジネスルール一覧

| 項目 | ルール |
|------|--------|
| 構成 | 男3・女3＝6人で1グループ |
| エリア | 恵比寿 / 池袋 / 銀座（初期） |
| 枠 | 運営が「日時×エリア」を用意。ユーザーは応募のみ |
| チャット | **無し**（連絡は成立後のLINE通知のみ） |
| 会場 | 運営が手動決定 → 店名/予約URL/予約名/集合 を6名へ通知 |
| 料金 | 男¥2,000（初回無料）/ 女無料 / 不成立非課金 |
| 評価 | イベント後に同席者を相互評価（1-5・任意コメント・匿名集計） |
| 優良バッジ | 平均評価≥4.0 かつ 評価件数≥5 かつ 参加≥2回 で付与 |
| 通知 | 成立 / 会場確定 / 評価のお願い / バッジ付与 を LINE push（現状モック） |
