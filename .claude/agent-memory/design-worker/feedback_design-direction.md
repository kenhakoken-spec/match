matching-app のビジュアル方向は **「"AIっぽさ"を排した、温かく編集的（editorial）」** で固定。

- **避ける（=AIっぽい/量産SaaS, FAIL判定対象）:** 紫〜青グラデ、ネオン/ビビッド原色、絵文字過多、18px超の大角丸+強い影の「ぷっくり」感、巨大ヒーロー+3カラム特徴のテンプレLP構図、煽りマーケコピー（運命の出会い/今すぐ無料/業界No.1）。
- **目指す:** 生成りのオフホワイト地 + 焦げ茶インク、テラコッタ（accent #C2703D / hover #A85638）を主役1アクセント、深緑（#5E7A57）を補助差し色、写真を主役にする余白、見出し明朝(serif)×本文ゴシック(sans)、影は最小・細ボーダー主体、誠実で機能的なコピー。
- 一言基準: 「雑誌の特集ページに置けるか。SaaS/AIのLPっぽくないか。」

追加要件（本人認証/決済/評価/バッジ/限定）でも方針は**不変**。新要素の扱い（design-system.md §4.7 / §8 で確定）:
- **優良バッジ** = くすんだ真鍮(trust系 #8A6D3B/#D8C39A/#F3EAD8)の静かな pill。金ピカ・グラデ・光彩は禁止。名前横に控えめに。
- **決済の初回無料** = 静かに主役(accent/100地・🎁は1つ)。販促トーン(「女性無料!」「今だけ0円!!」)禁止。「成立後・不成立は非課金」を必ず明記。
- **本人認証の却下** = state/warn(注意)であって danger(赤エラー)にしない。責めず再提出を促す。PII削除を明示。
- **参加条件NG枠** = 赤エラーにせず淡色+破線+事実の理由(muted)。落ち度ではない扱い。
- **相互評価** = 誠実・任意・匿名性(個別開示なし)を明示。ランキング/競争/FOMO演出は禁止。

**Why:** 殿が master_plan §2 で明示した世界観であり、S0設計タスクで「デザイン方向が量産SaaS的・AIっぽい」ことは明確なFAIL条件。確定済みデザイントークン全量は `docs/design/design-system.md`（§0原則・§4.7新コンポーネント・§7早見表・§8禁止リスト）。
**How to apply:** 配色・コンポーネント・コピーの提案時は design-system.md の §0/§4.7/§8 禁止リストに毎回照合。新色追加時もテラコッタ1アクセント原則と低彩度ニュートラルを崩さない。WCAG AA（小文字on accentは #A85638 に上げる、状態は色のみに依存しない）。関連: [[project-product-constraints]]
