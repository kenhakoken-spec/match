# S8 追加要望（正典）— マーケ強化・AI認証・多軸評価・ドタキャン罰金・会場候補

殿の追加指示（2026-05-31）。master_plan の上位に位置する確定要件。S8として実装する。

## 要望1: 「見えるけど できない」マーケ設計（プレビュー）
- **未ログイン/未登録でも、枠一覧＋枠詳細まで見える**。
- 枠詳細では**参加者の「すごさ」が見える**: 職種（occupation）・年代・評価★（多軸平均）・優良バッジ。
- **ただし個人特定情報は伏せる**: 氏名/表示名・顔写真・LINE は未登録には出さない（年代・職種・評価のみ＝匿名サマリ）。
- 応募ボタンは未登録だと「**登録して参加**」に変わり、タップで**登録導線へ誘導**（＝見えるが予約はできない）。
- 全画面を見直し、「まず見える→制限→登録/本人確認を促す」の原則を徹底（応募ゲートで弾くのでなく、見せてから促す）。

## 要望2: ゲートのAI（Haiku）自動化 ＋ 会場候補レコメンド
- **本人認証の一次判定をAI(Haiku)で**: 身分証画像から ①18歳以上か ②顔写真の有無 ③記載の読取 を判定し、**OK/要確認/NG** を返す。
  - 運用: **AI一次判定→明白なOKは自動承認、グレーのみ運営が確認**（[[always-approve-autonomy]] のAI委任方針）。
  - 出会い系規制の年齢確認責任は重いので、判定根拠を必ず記録（監査）。実Haiku未接続時はモック判定。
- **会場候補レコメンド**（飲食店予約は人間=殿がやる前提。候補出しと通知を自動化）:
  - 成立した枠に対し、エリア×人数(6名)で**合コン向きの店候補**をリストアップ。
  - 各候補に **食べログ点数・Google点数** を併記、合コン向き度でソート。
  - 候補が揃ったら**運営へ通知**（殿が選んで予約→会場入力）。

## 要望3: 限定は「イベント（枠）ごと」
- **誰でもOKの会／20代限定の会／優良バッジ限定の会** を枠単位で混在。← 既存 Slot 設計（minAge/maxAge/requiresBadge が null可）で対応可能。**schema変更不要**。
- **初期seedは 水・金・土 の 19:30集合 の枠ばかり**にする（恵比寿/池袋/銀座）。当初は「誰でもOK」中心。
- **リリース前モード**: ユーザーが集まるまで本稼働しない。**「リリースをお待ちください」画面**を用意し、フラグ（env: `RELEASE_MODE=waiting|open`）で全体を待機表示に切替可能（最初から有効である必要はないが画面は作る）。

## 要望4: 多軸評価（バチェラーデート参考）
- ☆1つでなく**3軸**で評価:
  - **また会いたい** / **会話（盛り上がり）** / **マナー（誠実さ）** 各 ★1〜5。
  - 任意コメントは従来通り。
- 集計: 3軸それぞれの平均＋総合平均を Profile に保持。優良バッジ判定は総合平均（≥4.0）で。

## 要望5: ドタキャン防止（当日キャンセル罰金 ¥5,000）
- **当日キャンセル/無断欠席(no-show)は罰金 ¥5,000**。
- 自動判定: イベント後の評価で、同席者が対象者に「**来なかった**」を選べる。**2人以上**が選んだら no-show 確定。
- 確定したら**Stripeで¥5,000を自動課金**（登録カードへ）。誤報防止に2人以上を条件。
- Payment に種別（participation / no_show_penalty）を持たせる。

## schema 影響（S8基盤workerが拡張）
- **Profile**: `occupation`（職種）追加。多軸評価集計（`scoreAgainAvg`/`scoreTalkAvg`/`scoreMannerAvg` + 既存 ratingAvg=総合）。
- **Rating**: 単一 score → **3軸**（scoreAgain/scoreTalk/scoreManner）。+ `noShow`(bool, この評価者が対象を「来なかった」と報告)。
- **Payment**: `type`（participation | no_show_penalty）。penalty額¥5000。
- **VenueCandidate**（新規）: slotId/店名/URL/tabelogScore/googleScore/fitScore（合コン向き度）/status。
- **IdentityVerification**: AI判定の記録（`aiVerdict` ok|review|ng, `aiReason`）。
- 限定枠（要望3）は既存 Slot で対応＝変更不要。

## 公開（プレビュー）DTO
- `PublicSlotDTO` / `PublicMemberDTO`: 未認証に返す。**職種・年代・多軸評価★・優良バッジは含む / 氏名・写真・lineUserId・生年月日(年代のみ可)は含まない**。

## マーケ文言の方針
- 「登録して参加」「あと2名で成立」等、**事実ベースで背中を押す**。誇大・煽りはしない（design-system §8 維持）。
